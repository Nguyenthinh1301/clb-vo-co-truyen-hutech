Account Model - Mô hình Tài khoản

Cấu trúc Tài khoản Người dùng

id String (UUID)
Mã định danh duy nhất của tài khoản
username String
Tên đăng nhập (tối thiểu 3 ký tự)
email String
Địa chỉ email hợp lệ
password String (Hashed)
Mật khẩu đã được mã hóa (tối thiểu 8 ký tự)
fullName String
Họ và tên đầy đủ
phoneNumber String
Số điện thoại liên lạc
dateOfBirth Date
Ngày sinh
role String (Enum)
Vai trò: 'student', 'instructor', 'admin'
membershipStatus String (Enum)
Trạng thái thành viên: 'active', 'inactive', 'pending', 'suspended'
joinDate Date
Ngày gia nhập câu lạc bộ
lastLoginDate Date
Lần đăng nhập cuối cùng
profileImage String (URL)
Đường dẫn ảnh đại diện

Phương thức Tài khoản

  • createAccount(userData)
    Tạo tài khoản mới với dữ liệu người dùng
  • validateLogin(username, password)
    Xác thực thông tin đăng nhập
  • updateProfile(userId, updateData)
    Cập nhật thông tin hồ sơ người dùng
  • changePassword(userId, oldPassword, newPassword)
    Thay đổi mật khẩu tài khoản
  • resetPassword(email)
    Đặt lại mật khẩu qua email
  • deactivateAccount(userId)
    Vô hiệu hóa tài khoản
  • getUserByEmail(email)
    Tìm người dùng theo địa chỉ email
  • getUserById(userId)
    Lấy thông tin người dùng theo ID
  • updateMembershipStatus(userId, status)
    Cập nhật trạng thái thành viên
  • generateAuthToken(userId)
    Tạo token xác thực cho người dùng

Quy tắc Xác thực

  • Username: Độ dài 3-20 ký tự, chỉ chứa chữ cái, số và dấu gạch dưới
  • Email: Định dạng email hợp lệ
  • Password: Tối thiểu 8 ký tự, bao gồm chữ hoa, chữ thường, số và ký tự đặc biệt
  • Phone Number: Định dạng số điện thoại Việt Nam (10-11 số)
  • Full Name: Không được để trống, độ dài tối đa 100 ký tự
  • Date of Birth: Phải từ 16 tuổi trở lên

Tính năng Bảo mật

  • Mã hóa mật khẩu bằng bcrypt
  • Token JWT cho xác thực phiên
  • Giới hạn số lần đăng nhập sai
  • Xác thực email khi đăng ký
  • Đặt lại mật khẩu qua email an toàn
  • Phân quyền theo vai trò (RBAC)
  • Ghi log các hoạt động quan trọng

Ví dụ Sử dụng Mô hình

1. Xử lý Đăng ký

// Trong file dang-ky.html - JavaScript xử lý form đăng ký function handleRegistration(formData) { const userData = { id: generateUUID(), username: formData.username, email: formData.email, password: hashPassword(formData.password), fullName: formData.fullName, phoneNumber: formData.phoneNumber, dateOfBirth: formData.dateOfBirth, role: 'student', membershipStatus: 'pending', joinDate: new Date().toISOString(), lastLoginDate: null, profileImage: '/assets/images/default-avatar.jpg' }; // Validate dữ liệu if (validateAccountData(userData)) { const newAccount = createAccount(userData); localStorage.setItem('registeredUser', JSON.stringify(newAccount)); window.location.href = 'dang-nhap.html'; } }

2. Xử lý Đăng nhập

// Trong file dang-nhap.html - JavaScript xử lý đăng nhập function handleLogin(username, password) { const loginResult = validateLogin(username, password); if (loginResult.success) { // Cập nhật lastLoginDate const updatedUser = { ...loginResult.user, lastLoginDate: new Date().toISOString() }; // Lưu token và thông tin user const authToken = generateAuthToken(updatedUser.id); localStorage.setItem('authToken', authToken); localStorage.setItem('currentUser', JSON.stringify(updatedUser)); // Chuyển hướng tùy theo role if (updatedUser.role === 'admin') { window.location.href = 'dashboard.html'; } else { window.location.href = '../../index.html'; } } else { showError('Thông tin đăng nhập không chính xác'); } }

3. Cập nhật Profile

// Trong file profile.html - JavaScript cập nhật thông tin function updateUserProfile(userId, updateData) { const validationResult = validateProfileUpdate(updateData); if (validationResult.isValid) { const updatedProfile = updateProfile(userId, updateData); localStorage.setItem('currentUser', JSON.stringify(updatedProfile)); showSuccess('Cập nhật thông tin thành công!'); } else { showError(validationResult.errors); } }
